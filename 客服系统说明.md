# 肤润康 AI 客服系统说明

> 本文档面向客服/运营/管理员，说明系统的回复机制、邮件通知规则、客户端等待提示、对话记录（日志），
> 以及后台（客服工作台）可以配置的全部项目。技术实现细节可参考 `README.md`。

## 一、三种工作模式

系统支持三种工作模式，管理员可在客服工作台顶部的模式切换器中随时切换（客服/非管理员只能查看当前模式，不能修改）：

| 模式 | 客户提问后的处理方式 |
| --- | --- |
| **全AI模式** | AI 自动检索题库，命中则直接把答案发给客户；未命中则转人工。 |
| **全人工模式** | AI 完全不参与，所有问题都进入人工队列，由客服手动查看并回复。 |
| **人机协同模式** | AI 先在题库中检索：命中则生成建议回复（客服可见、可编辑），并进入自动发送倒计时；未命中则不生成建议，直接转人工。 |

**未命中题库的判定规则（对所有模式统一）**：只要题库/产品资料里没有明确依据能回答这个问题，就一律判定为"未命中"，AI 不允许凭常识编造或延伸回答，必须转人工处理。

## 二、转人工邮件通知机制

只要一条提问需要人工处理，系统就会**立即**发一封邮件通知客服/管理员，邮件内容包含：产品、工作模式、转人工原因、客户的原始提问、对话编号。具体触发规则：

| 模式 | 是否发邮件通知 |
| --- | --- |
| 全人工模式 | **每一条新问题**都会发邮件（因为该模式下 AI 完全不作答） |
| 全AI模式 | 仅当题库**未命中**时发邮件（命中时 AI 已直接回复，无需打扰人工） |
| 人机协同模式 | 仅当题库**未命中**时发邮件（命中时 AI 已生成建议、进入自动发送倒计时，无需打扰人工） |

除了这种"单条即时提醒"，后台还可以额外开启**待处理队列定时提醒**和**每日数据日报**这两个周期性邮件（见下文"后台可配置项"）。

## 三、客户端等待提示逻辑

客户在网页端提问后，聊天气泡会显示动态的等待提示（文字后面带跳动的小圆点），具体文案会根据"题库是否命中"分成两条不同的分支，全程不会出现互相矛盾的措辞：

**分支 A：题库命中（全AI直接答完 / 协同模式命中、AI 正在准备自动发送）**

- 提问后立即显示：「AI 正在分析您的问题...」
- 拿到结果后（命中）：全AI模式直接显示最终答案；协同模式显示「AI 正在思考您的问题，请稍候...」
- 等待超过 **10 秒**仍未有最终结果：显示「AI 思考时间较长，请耐心等待」（全程不出现"人工客服"字样，避免误导客户）
- 等待超过约 10 分钟仍无结果（极端情况）：显示「AI 暂时无法给出回复，请稍后重新提问，或联系人工客服协助处理。」

**分支 B：题库未命中 / 全人工模式（需要真人处理）**

- 拿到结果后显示：「已收到您的问题，正在为您转接人工客服，请稍候...」
- 等待超过 **10 秒**客服仍未回复：显示「人工客服正忙，请您稍后查看回复，感谢您的耐心等待～」
- 等待超过约 10 分钟仍无结果（极端情况）：显示「客服暂时繁忙，请稍后重新提问或换个方式联系我们。」

两条分支背后都是同一套轮询机制：客户端每 3 秒查询一次对话状态，客服/AI 一旦给出最终回复，会自动替换成实际内容，客户不需要手动刷新页面。

## 四、对话记录（日志）

每一条客户提问都会永久记录在数据库（`data/qa.db`，已通过 Docker volume 持久化，重启/更新容器不会丢失），包含以下信息：

- 客户所属会话（自动分配的"访客编号"）、提问内容、提问时间
- 本次使用的工作模式、咨询的产品
- 题库检索结果：是否命中、命中的题库问题/产品资料段落、相似度分数
- 处理状态：待处理 / 已回复，以及最终回复内容、由谁（AI 自动发送 / 具体客服）回复、回复时间
- 客户端 IP（仅供参考，不作为区分客户的唯一依据）

客服/管理员可在工作台的会话列表中按客户查看完整的连续对话记录和检索命中详情，无需额外操作。

## 五、后台（客服工作台）可配置项一览

所有配置项均可在客服工作台右上角「工作台设置」面板中修改，标注"仅管理员"的项目普通客服账号不可见/不可修改。

| 配置项 | 入口 | 权限 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 全局工作模式 | 顶部模式切换器 | 仅管理员 | 全AI模式 | 全AI / 全人工 / 人机协同三选一，立即生效 |
| 人机协同自动发送超时 | 工作台设置 | 仅管理员 | 5 秒 | 1-300 秒，超时未被客服认领则自动把 AI 建议发给客户 |
| SMTP 发信配置 | 工作台设置 → SMTP 发信配置 | 仅管理员 | 无（回退到 `.env`） | 发信服务器、端口、账号、密码、发信人、TLS/SSL，支持一键发送测试邮件；配置存数据库，优先于 `.env` |
| 邮件提醒收件人 | 工作台设置 → 邮件提醒收件人 | 仅管理员 | `.env` 中 `NOTIFY_EMAIL_TO` | 支持填多个邮箱，用英文逗号分隔；上述"即时提醒""定时提醒""每日日报"都发到这里 |
| 待处理队列定时提醒 | 工作台设置 → 待处理队列定时提醒 | 仅管理员 | 关闭，30 分钟 | 开启后只要队列不为空，就按设定分钟数（1-1440）周期发汇总邮件 |
| 每日数据日报 | 工作台设置 → 每日数据日报 | 仅管理员 | 开启，每天 09:00（香港时间） | 每天到点推送前一天的咨询用户数、总对话条数、转人工请求次数 |
| 客服账号管理 | 工作台设置 → 客服账号管理 | 仅管理员 | - | 新增客服/管理员账号 |
| 修改自己的密码 | 顶部账号菜单 | 所有账号 | - | 客服和管理员均可修改自己的登录密码 |

## 六、题库/产品资料更新方式

- 有 Excel 题库的产品（如杜鹃花酸乳霜）：直接更新对应的 Excel 文件，服务重启后自动重新建立检索索引。
- 用说明文档做知识源的产品（如 10% 尿素护手霜）：直接更新对应的 Markdown 说明文档（如 `urea_hand_cream_info.md`），服务重启后自动按二级标题重新切片、重新建立检索索引。文档里没有明确写到的内容，AI 一律不会回答，会转人工处理。
