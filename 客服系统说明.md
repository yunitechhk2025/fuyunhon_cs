# 澳洲肤润康 AI 客服系统说明

> 本文档面向客服/运营/管理员，说明系统的回复机制、邮件通知规则、客户端等待提示、对话记录（日志），
> 以及后台（客服工作台）可以配置的全部项目。技术实现细节可参考 `README.md`。

## 一、三种工作模式

系统支持三种工作模式，管理员可在客服工作台顶部的模式切换器中随时切换（客服/非管理员只能查看当前模式，不能修改）：

| 模式 | 客户提问后的处理方式 |
| --- | --- |
| **全AI模式** | AI 自动检索题库，命中则直接把答案发给客户；未命中则转人工。 |
| **全人工模式** | AI 完全不参与，所有问题都进入人工队列，由客服手动查看并回复。 |
| **人机协同模式** | AI 先在题库中检索：命中则生成建议回复（客服可见、可编辑），并进入自动发送倒计时；未命中则不生成建议，直接转人工。 |

**未命中题库的判定规则（对所有模式统一）**：只要题库/产品资料里没有明确依据能回答这个问题，就一律判定为"未命中"，AI 不允许凭常识编造或延伸回答，必须转人工处理。

**例外1：单纯打招呼**（"你好""在吗""hi"等，不带具体问题）不算"未命中"——任何模式下都会由 AI 直接回一句问候语、引导客户说出具体问题，不会转人工、也不会发邮件提醒。但如果打招呼的同时带了具体问题（比如"你好，能天天用吗"），会正常按题库检索流程处理，不受此例外影响。

**例外2：与产品咨询完全无关的闲聊/荒谬提问，或不构成真实客服需求的挑逗性提问**（比如"吃饭了嘛""这产品对国家安全有危害吗""哪里买到假货"）：AI 会先做一次轻量判断，只有**非常确信**这条消息完全不构成真实客服需求（纯闲聊寒暄、荒谬/挑衅性提问，或字面上就是问"哪里能买到假货本身"这类恶作剧式提问）时，才会直接回复一句引导语，不落库、不进人工队列、不发邮件提醒；只要有一点点可能是在问产品本身、成分、使用方法、适用人群、购买渠道、防伪辨别（例如"怎么辨别真伪""在哪买正品才不会买到假货"）等内容，就会判定为"相关"，正常走检索/转人工流程——宁可放过、不可错判，避免真实问题被悄悄漏单。管理员可在工作台的「无关闲聊自动过滤」中随时关闭这个判断，一键退回"全部按正常问题处理"。

**例外3：客户直接说"转人工""人工客服""找客服"等明确要求转接真人的话**：这不是一句需要检索题库或让 AI 判断的问题，系统会在"无关闲聊"判断之前就把它识别出来（避免像例外2那样被误判成"与产品无关"），直接跳过题库检索、进入转人工状态。因为"转人工"这几个字本身没有实际咨询内容，客户端会先请客户补充说明一次具体想咨询的问题（必填），再选填一个邮箱，详见下文"客户端等待提示逻辑"。

## 二、转人工邮件通知机制

系统**不会**在一条提问刚进入人工队列时就立即发邮件——不管是题库未命中，还是全人工模式下的任意问题，一律先只把这条对话实时广播给客服工作台（客服在工作台随时能看到待处理队列），**不发邮件**。

邮件通知只有一个触发条件：**客户在等待过程中主动留下了合法格式的邮箱**（无论是分支 B 的"留邮箱"入口，还是分支 C 补充问题时选填的邮箱），此时才会真正发一封邮件通知客服，内容包含：产品、工作模式、客户提问、客户邮箱、对话编号。客户不填邮箱，则完全不会触发任何邮件。

**全人工模式额外例外**：即使客户在全人工模式下留了邮箱，也**不会**发邮件——该模式下客服本就需要一直盯着工作台处理每条对话，不需要邮件提醒；客户留的邮箱依然会保存下来，在工作台里可见，方便客服直接联系客户，只是不会触发邮件这个动作。

除了这种"客户留邮箱"触发的即时提醒，后台还可以额外开启**待处理队列定时提醒**和**每日数据日报**这两个周期性邮件（见下文"后台可配置项"），这两个周期性邮件不受上面"全人工模式不发邮件"的例外影响。

## 三、客户端等待提示逻辑

客户在网页端提问后，聊天气泡会显示动态的等待提示（文字后面带跳动的小圆点），具体文案会根据"题库是否命中"分成两条不同的分支，全程不会出现互相矛盾的措辞：

**分支 A：题库命中（全AI直接答完 / 协同模式命中、AI 正在准备自动发送）**

- 提问后立即显示：「AI 正在分析您的问题...」
- 拿到结果后（命中）：全AI模式直接显示最终答案；协同模式显示「AI 正在思考您的问题，请稍候...」
- 等待超过 **10 秒**仍未有最终结果：显示「AI 思考时间较长，请耐心等待」（全程不出现"人工客服"字样，避免误导客户）
- 之后网页会持续轮询，不设置"等太久就放弃"的兜底提示——AI 的反馈不会超过这个量级，没必要再单独设置一个更长的超时文案。

**分支 B：题库未命中（全AI / 协同模式）——会有"留邮箱"提示**

- 拿到结果后显示：「已收到您的问题，正在为您转接人工客服，请稍候」——这条消息会一直保留在对话记录里，**不会被后续提示覆盖或删除**。
- 等待超过 **10 秒**客服仍未回复：在上面那条消息**下方新增一条**消息，显示「人工客服正忙，留下你的電郵，客服稍後通過電郵回復您」，并就地提供一个邮箱填写框和「提交」按钮。客服最终的回复会写入这条新消息里。
- 如果客服在 **10 秒内**就回复了（还没来得及弹出上面的留邮箱提示），回复会直接替换掉「已收到您的问题…」这条消息——相当于"请稍候"自然过渡成了真正的答案，没有额外新消息。

**分支 B'：全人工模式的任意问题——完全没有"留邮箱"提示**

- 拿到结果后同样显示：「已收到您的问题，正在为您转接人工客服，请稍候」——这条消息同样**一直保留在对话记录里**，之后就只是安静持续轮询，**不会**在 10 秒后弹出"人工客服正忙/留邮箱"这套提示。因为全人工模式下客服本就要一直盯着工作台处理每条对话，问题已经直接进了待处理队列，不需要客户额外留邮箱这套机制。
- 客服最终在工作台回复后，答案会**另起一条新消息**展示，同样不会覆盖或替换掉「已收到您的问题…」这条记录。

**分支 C：客户直接说"转人工"之类明确要求转接真人（见上面例外3）**

- 不需要等 10 秒、也不会先经历"AI 正在思考"的过程，网页会立即显示「好的，请您简单描述一下想咨询的具体问题，我们马上为您转接人工客服～」，并提供输入框：
  - **想咨询的问题**（必填）：因为"转人工"这几个字本身不是一句有内容的提问，客服光看这个不知道要处理什么，所以要请客户补充说明；客户填写并提交后，会用这段内容替换掉数据库里原来"转人工"这句占位提问，客服在工作台看到的就是客户真正想问的问题。
  - **邮箱**（选填，**全人工模式下不显示这个输入框**）：留下真实格式的邮箱后客服可以直接通过邮件回复；不填邮箱不影响问题内容的提交更新，客服仍能在工作台的待处理队列里实时看到这条对话。当前工作模式是全人工模式时，这套流程只会展示"问题"这一个输入框，因为该模式下留邮箱完全没有意义。

分支 B 的留邮箱入口、分支 C 的补充问题入口（全人工模式除外），在"邮箱"这一部分的规则是一样的：

- 客户填写邮箱并点击提交后，后端会先校验是不是**真实格式的邮箱**（形如 `xxx@xxx.xxx`），格式不对会提示错误、拒绝提交；只有格式合法才会真正发一封邮件通知客服，内容包含客户提问、客户邮箱（如果留了）、对话编号，客服可直接通过邮件回复客户；提交成功后输入框会消失，替换成"已将您的问题更新给人工客服"的确认文字。
  - **转人工必须要客户填写（合法格式的）邮箱才会真正发邮件通知客服**——客户如果不填邮箱，则完全不会触发这条邮件通知；但无论是否填了邮箱，客服都能在工作台的待处理队列里实时看到这条对话。
  - **全人工模式下完全没有留邮箱这套机制**（无论是分支 B' 还是分支 C）：问题直接进入待处理队列，客户端不会展示任何邮箱相关的输入框或提示，也不会有对应的邮件通知（见上文"二、转人工邮件通知机制"）。
  - 客户提交后，页面仍会继续轮询等待客服的最终回复；一旦客服在网页端也回复了，还是会正常替换成实际内容。
- 涉及转人工的情况在 10 秒时已经直接进入了"留邮箱 / 补充问题"这套转人工机制（全人工模式则是直接安静轮询），客户随时可以在那个入口留下联系方式；不设置额外的"等太久就放弃"的兜底提示，页面会一直保留输入框、持续轮询，直到客服给出回复为止。

以上分支背后都是同一套轮询机制：客户端每 3 秒查询一次对话状态，客服/AI 一旦给出最终回复，会自动替换成实际内容，客户不需要手动刷新页面。

**刷新页面 vs 关闭页面**：客户如果只是刷新网页（F5/下拉刷新），聊天记录会自动恢复——之前问过的问题和已经收到的回复都会重新显示出来，还没等到回复的问题也会接着轮询，不需要重新问一遍；但如果客户把这个页面/标签页整个关掉再重新打开，则会当成一次全新的对话，不会带着之前的聊天记录，同时也会被识别成一个新的"访客"（详见下文"如何区分不同客户"）。

## 四、客服工作台的回复框

**回复框只在真正需要人工处理时才会出现**：

| 情况 | 是否显示回复框 |
| --- | --- |
| 全AI模式命中题库、已直接回复客户 | 不显示（已经回复完毕，没有可编辑的内容） |
| 人机协同模式命中题库、AI 已生成建议并进入自动发送倒计时 | **不显示**：这类问题不算"转人工"，AI 会在倒计时结束后自动把建议发给客户，不需要客服人工介入 |
| 题库未命中（全AI / 协同模式） | 显示：需要客服根据实际情况人工回复 |
| 全人工模式的任意问题 | 显示：该模式下 AI 从不直接作答，每条问题都需要客服手动回复 |
| 客户主动说"转人工"（见上面例外3 / 分支C） | 显示：客户明确要求转接真人，需要客服人工回复 |

也就是说，"回复框"只在对话真正需要客服手动处理（未命中题库 / 全人工模式 / 客户主动要求转人工）时才会出现；AI 已经处理完（直接回复，或已生成建议等待自动发送）的对话不会有回复框，避免客服误以为所有对话都需要手动回复。

回复框只有一个「发送回复」按钮，客服随时可以直接输入内容发送给客户；发送成功后这条对话变为"已回复"状态，回复框自动消失。（之前有一套"认领此问题 / 释放认领"的多客服协作机制，只在多个客服同时在线、可能抢着处理同一个对话时才有意义，已按需去掉，回复流程更简单。）

**回复框内容不会被意外清空**：工作台会每隔几秒（或有新消息推送时）自动刷新对话列表，客服正在输入但还没点击"发送回复"的内容会被自动保存在草稿箱里，刷新时优先用草稿箱的内容填充输入框，不会因为页面刷新而把已经打的字冲掉；发送成功后草稿会被清空。另外，只要客服的光标还停留在回复框里（包括用中文输入法打字、还没上屏的拼音候选词阶段），页面会**暂停**这类自动刷新，避免刷新把输入框整个换成新的，导致还没上屏的候选词凭空消失；等客服点击别处（输入框失去焦点）后，页面会立刻补一次刷新，追平这段时间内的新消息。

## 五、对话记录（日志）

每一条客户提问都会永久记录在数据库（`data/qa.db`，已通过 Docker volume 持久化，重启/更新容器不会丢失），包含以下信息：

- 客户所属会话（自动分配的"访客编号"）、提问内容、提问时间
- 本次使用的工作模式、咨询的产品
- 题库检索结果：是否命中、命中的题库问题/产品资料段落、相似度分数
- 处理状态：待处理 / 已回复，以及最终回复内容、由谁（AI 自动发送 / 具体客服）回复、回复时间
- 客户端 IP（仅供参考，不作为区分客户的唯一依据）
- 客户主动留下的邮箱（仅在转人工场景下客户自愿填写、且通过格式校验时才会有；未填写或格式不合法则为空）

数据库里存的提问时间/最近活跃时间是 UTC 时间（服务器时间），但客服工作台展示给客服看的时间（对话列表、每条提问旁边的时间）都会自动换算成**北京时间**（UTC+8）显示，不需要客服自己心算时差。

客服/管理员可在工作台的会话列表中按客户查看完整的连续对话记录和检索命中详情；如果客户留下了邮箱，对应问题的元信息里会额外显示一个「📧 邮箱」标签，方便客服直接通过邮件回复，无需额外操作。

**如何区分不同客户（访客编号是怎么来的）**：系统不需要客户登录/留任何身份信息，靠的是浏览器里的一个随机会话标识（`session_id`）——客户第一次打开客服页面时，浏览器会自动生成一个随机 id 并记住它，之后这个浏览器标签页里问的每一个问题都带着同一个 id，后端就知道这些问题都是"同一个人"问的。工作台看到的「访客1」「访客2」……这个编号，就是按不同 `session_id` 第一次提问的时间先后顺序自动排出来的，纯粹是给客服看的一个好记的代号，跟真实身份无关。客户端 IP 也会记录下来，但只作参考展示（同一个 IP 可能是好几个人共用的网络，不能直接当成"同一个人"的依据）。

这个"记住客户"的标识存在浏览器的临时存储里，跟随"标签页"的生命周期：客户刷新页面还是同一个标识（历史记录能恢复、workbench 看到的还是同一个访客编号）；但只要客户把这个标签页关掉，下次重新打开就会生成一个全新的标识，会被当成一个新的访客编号，即使实际上是同一个人。

## 六、后台（客服工作台）可配置项一览

所有配置项均可在客服工作台右上角「工作台设置」面板中修改，标注"仅管理员"的项目普通客服账号不可见/不可修改。

| 配置项 | 入口 | 权限 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 全局工作模式 | 顶部模式切换器 | 仅管理员 | 全AI模式 | 全AI / 全人工 / 人机协同三选一，立即生效 |
| 人机协同自动发送超时 | 工作台设置 | 仅管理员 | 5 秒 | 1-300 秒，超时客服还没发送回复则自动把 AI 建议发给客户 |
| SMTP 发信配置 | 工作台设置 → SMTP 发信配置 | 仅管理员 | 无（回退到 `.env`） | 发信服务器、端口、账号、密码、发信人、TLS/SSL，支持一键发送测试邮件；配置存数据库，优先于 `.env` |
| 邮件提醒收件人 | 工作台设置 → 邮件提醒收件人 | 仅管理员 | `.env` 中 `NOTIFY_EMAIL_TO` | 支持填多个邮箱，用英文逗号分隔；上述"即时提醒""定时提醒""每日日报"都发到这里 |
| 待处理队列定时提醒 | 工作台设置 → 待处理队列定时提醒 | 仅管理员 | 关闭，30 分钟 | 开启后只要队列不为空，就按设定分钟数（1-1440）周期发汇总邮件 |
| 每日数据日报 | 工作台设置 → 每日数据日报 | 仅管理员 | 开启，每天 09:00（香港时间） | 每天到点推送前一天的咨询用户数、总对话条数、转人工请求次数 |
| 无关闲聊自动过滤 | 工作台设置 → 无关闲聊自动过滤 | 仅管理员 | 开启 | AI 判断消息是否与产品咨询完全无关，非常确信才跳过转人工；关闭后退回"全部按正常问题处理" |
| 客服账号管理 | 工作台设置 → 客服账号管理 | 仅管理员 | - | 新增客服/管理员账号 |
| 修改自己的密码 | 顶部账号菜单 | 所有账号 | - | 客服和管理员均可修改自己的登录密码 |

## 七、题库/产品资料更新方式

- 有 Excel 题库的产品（如杜鹃花酸乳霜）：直接更新对应的 Excel 文件，服务重启后自动重新建立检索索引。
- 用说明文档做知识源的产品（如 10% 尿素护手霜）：直接更新对应的 Markdown 说明文档（如 `urea_hand_cream_info.md`），服务重启后自动按二级标题重新切片、重新建立检索索引。文档里没有明确写到的内容，AI 一律不会回答，会转人工处理。
